# Журнал изменений

## 1.1.0

### Интерфейс
- Веб-интерфейс, локализация и диагностика теперь работают только с `chromium`, `ungoogled-chromium` и `librewolf`: удалены подписи `Wayfern`, поток подтверждения условий, старые значки движков и устаревшая тестовая поверхность инструментов runtime.

### Документация
- README, вики, операторское руководство по релизу и тесты качества документации лаунчера синхронизированы под финальный переход на `chromium` + `ungoogled-chromium` + `librewolf`; рабочая документация и скрипты больше не допускают возврат `Wayfern` или `Camoufox`.


### Ядро
- Семейство Chromium runtime переведено на Chromium: активный адаптер Wayfern удален из путей лаунчера, runtime, CLI и тестов, а LibreWolf сохранен как единственный runtime семейства Firefox.
- Управляемый лаунчером Chromium runtime теперь получает артефакты из официального хранилища снимков Chromium и продолжает готовить аргументы запуска профиля, policy-расширения и переопределения локали identity без Wayfern-специфичных веток.
- Управляемое лаунчером семейство Chromium теперь поддерживает два открытых Chromium runtime: `chromium` и `ungoogled-chromium`, с единым путем подготовки запуска и области расширений, без compatibility-only алиаса обратно в `Wayfern`.
- Путь загрузки Windows для управляемого runtime движков усилен fallback-механизмом: если `curl.exe` получает connect-timeout (`curl: (28)`) или длительный zero-byte старт на GitHub-hosted asset, загрузка автоматически переключается на встроенный поток `reqwest` без отмены запуска профиля.

### Безопасность
- Полный отказ от Wayfern выполнен без обратной совместимости: удалены шлюз подтверждения условий, команды подтверждения в лаунчере и compatibility-only путь начальной инициализации для wayfern.
- Встроенные профили Chromium по умолчанию и инструменты runtime теперь используют только `engine:chromium`, без скрытой нормализации старого идентификатора Wayfern.

### Профили
- Модель профиля и поверхности сессий/ runtime лаунчера теперь используют Chromium как единственный идентификатор движка семейства Chromium, сохраняя изоляцию профилей и текущую модель подготовки профиля.
- Профильные сценарии strict app-window и single-page теперь явно описаны как поведение семейства Chromium, без скрытой привязки к `Wayfern`.
