# Журнал изменений

## 1.1.7

### Профили и запуск
- Исправлена логика стартовой страницы профиля: пустое значение теперь корректно сохраняется и не должно самопроизвольно возвращаться после повторного открытия модалки.
- Для LibreWolf убран конфликт двойного старта (сессия + принудительная стартовая страница): при наличии восстанавливаемой сессии домашняя страница не форсируется.

### Расширения (profile-owned)
- Расширения переведены на профильную модель хранения для Chromium и LibreWolf: пакеты и метаданные хранятся внутри конкретного профиля.
- Синхронизация состояния двусторонняя: изменения в браузере (включение/выключение/удаление) отражаются в UI, а изменения из UI применяются в профиль.
- Удалены legacy-артефакты и конфликтные сценарии одновременной загрузки расширений.

### Chromium
- Стабилизирован запуск профильных расширений: устранены повторные lifecycle-срабатывания, из-за которых некоторые расширения вели себя как «впервые установленные».
- Для Dark Reader добавлена защита от повторного открытия welcome/help-вкладки и дополнительная диагностика применения патча.
- Для Dark Reader добавлен приоритет установки через `External Extensions` (где возможно), чтобы исключить повторный onboarding при dev/unpacked-запуске.

### LibreWolf
- Расширения устанавливаются как профильные `.xpi` в `engine-profile/extensions` вместо старой policy-схемы.
- Добавлена стабильная синхронизация состояния расширений по `extensions.json`.

### Логи и диагностика
- Снижена шумность логов по ожидаемым сетевым разрывам.
- Добавлены точечные launch-логи для отладки расширений, startup URL и handoff процессов.
