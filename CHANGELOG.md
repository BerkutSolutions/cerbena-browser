# Журнал изменений

## 1.0.12

### Core

- Исправлен запуск апдейтера и его предпросмотра: `--updater` и `--updater-preview` больше не распознаются как внешние `http/https`-ссылки, не требуют выбора профильной привязки, не активируют основной браузерный профиль и не ломаются на single-instance-механизме launcher-а.
- Усилена panic-frame рамка на Windows: overlay теперь привязывается к реальному окну браузера как owner-bound слой, двигает border/label/controls/menu пакетно через `DeferWindowPos`, мгновенно скрывается при потере или закрытии главного окна, больше не отстаёт так заметно при быстром resize и больше не сканирует процессы на каждом `8 мс` тике, что заметно снижает загрузку CPU.

### Wayfern

- Исправлен запуск `Wayfern`/`Chromium` после свежей загрузки: окно подтверждения в интерфейсе содержит прямую гиперссылку на `https://wayfern.com/terms-and-conditions`.
- Ужесточено подтверждение условий `Wayfern`: перед каждым запуском `Wayfern` интерфейс теперь заново запрашивает launcher-side статус принятия, повторно показывает предупреждение после `Отмены` и не считает согласие завершённым, пока backend не подтвердит актуальный статус.
- Исправлен ранний краш `Wayfern` на чистых профилях и локальных/установленных запусках: launcher теперь при подтверждении условий и перед стартом синхронизирует глобальный roaming-маркер `Wayfern\\license-accepted`, который сам движок ожидает записывать рядом со своим встроенным license-check, вместо того чтобы полагаться только на локальный `wayfern_tos_ack.json`.
- Для `Wayfern` на Windows launcher теперь предпочитает запуск через локальную копию `wayfern-browser.exe`, если пакет движка отдает только `chrome.exe`, чтобы обойти среды, где голое имя `chrome.exe` закрывается защитой/политикой сразу после старта без полезных crash-логов.
- Исправлен ложный цикл запуска `Wayfern` после принятия условий: launcher больше не передаёт `--accept-terms-and-conditions` в обычный запуск профиля, потому что этот флаг запускает одноразовый acceptance-flow и затем штатно завершает процесс вместо открытия браузера.
- Принятие условий `Wayfern` доведено до автоматического сценария: после согласия пользователя launcher сам тихо завершает vendor acceptance-flow для текущей версии движка и только потом выполняет обычный запуск профиля, без ручного второго старта.

### UI

- Для каждого профиля добавлена кнопка просмотра логов рядом с запуском: launcher хранит профильные события 24 часа и в модалке показывает историю запуска, текущие runtime-файлы и `Docker`-логи для контейнерной изоляции, если они доступны.
- Секция логов профиля теперь показывает движковые файлы `wayfern-debug.log`, `wayfern-stdout.log` и `wayfern-stderr.log`, а сама модалка ограничена по высоте и прокручивает только внутреннюю «консоль».

### Диагностика

- Исправлена ложная успешность запуска `Wayfern`/`Camoufox`: launcher больше не пишет `Browser launched successfully`, если браузер умер в первые секунды, а сохраняет в профильные логи ранний выход процесса.
- Для диагностики `Wayfern` launcher теперь пишет движковый debug-log в `profile/tmp/wayfern-debug.log`, логи `stdout/stderr` процесса и запускает `Wayfern` с рабочим каталогом из директории самого бинаря, а не из корня профиля.

### Updater

- Исправлен updater e2e/preflight: `published-updater-e2e.ps1` теперь корректно читает `lastError` из `app_update_store.json`, копирует в snapshot новые неигнорируемые файлы рабочего дерева и не считает штатный background handoff на статусе `applying` ошибкой до истечения таймаута.
- Для updater-driven selftest relaunch добавлен управляемый авто-выход по таймеру, чтобы проверочный перезапущенный launcher не зависал после успешного применения обновления.
