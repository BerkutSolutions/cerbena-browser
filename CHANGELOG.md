# Журнал изменений

## 1.0.15

### Профили

- Исправлено удержание языка интерфейса в `Wayfern` при split tunneling и внешнем IP другой страны: launcher-расширение принудительно задает `Accept-Language` из выбранной личности для HTTP/HTTPS-запросов, чтобы сайты не переключали интерфейс на язык VPN/IP.
- Исправлена совместимость `Real`-личности с реальным движком браузера: `Wayfern` и `Camoufox` больше не подменяют свой нативный `User-Agent` значением из launcher `WebView`, поэтому Google-вход и проверки безопасности получают более согласованный отпечаток браузера.
- Улучшено принудительное применение языка профиля в `Wayfern`: заголовок `Accept-Language` теперь формируется в более браузерном виде с приоритетами `q=`, чтобы сайты реже игнорировали выбранный язык и переключались по гео/IP.
- В профильной вкладке `VPN` добавлено зелёное уведомление для режима `Напрямую`, если глобальный маршрут всё равно перехватывает трафик через выбранный шаблон VPN/Proxy: Cerbena теперь явно показывает имя активного глобального маршрута и объясняет, что причиной являются глобальный VPN и/или блокировка трафика вне VPN.
- Ошибки запуска container-изоляции из-за `Docker Desktop` переведены из сырого backend-текста в понятные действия: если Docker не установлен, Cerbena открывает модалку со ссылкой на страницу загрузки; если Docker установлен, но runtime ещё не поднят, Cerbena просит запустить `Docker Desktop` и повторить запуск профиля.

### Безопасность

- Смягчено Chromium runtime hardening для `Wayfern`: из launch-флагов убраны отключения `AutofillServerCommunication`, `AutofillEnableAccountWalletStorage` и `PasswordManagerOnboarding`, которые мешали Google account/passkey-сценариям и вызывали ложные предупреждения о небезопасном или недоверенном приложении.
- Для `Wayfern` возвращен приоритет запуска исходного vendor-бинарника `chrome.exe` вместо launcher-копии `wayfern-browser.exe`, чтобы Windows и сайты с `passkey`/WebAuthn-подтверждением видели доверенный браузерный исполняемый файл, а не неподписанный alias.

### Стабильность

- Добавлена runtime-миграция старых `Wayfern`-установок: если в `installed-engines.json` остался устаревший путь на `wayfern-browser.exe`, launcher автоматически переключает профиль на соседний vendor `chrome.exe` и сохраняет исправленный `binary_path`.
- Усилен `process_tracking` для браузерных handoff-сценариев: launcher теперь удерживает сессию живой, если основной профильный процесс переспавнился и остались дочерние `chrome/wayfern/firefox/camoufox`-процессы из той же цепочки, чтобы не ронять `panic frame`, network stack и состояние профиля из-за ложного `stopped`.
- Исправлен импорт `Amnezia` `vpn://`-ключей для container runtime: launcher теперь отбрасывает пустые и пусто-кавыченные native-only поля `I1..I5` при генерации конфигурации `awg`, чтобы `amneziawg-go` не падал на строках вида `I2=''` и kill-switch не оставлял профиль без маршрута.
- Упрощено позиционирование `panic frame` на Windows: overlay-окна снова двигаются обычным Tauri-путём без жёсткой owner/z-order группировки, чтобы рамка снова отображалась, а окно браузера не залипало поверх остальных окон.
- Исправлена обработка `Amnezia` `last_config` из `vpn://`-payload: после подстановки DNS launcher теперь дополнительно санитизирует готовый конфиг перед записью в container runtime, чтобы строки вида `I2=''` не переживали путь через `last_config`.
- Для `panic frame` overlay-окна переведены в `always_on_top`, чтобы рамка снова рисовалась поверх окна браузера после отказа от старой нативной HWND-группировки.
- Версия приложения обновлена до `1.0.15` в desktop/runtime/UI fallback-строках, тестовых ожиданиях и release-facing метаданных.
