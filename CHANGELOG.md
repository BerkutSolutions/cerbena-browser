# Журнал изменений

## 1.0.11

### UI

- В `VPN`-форме пароль теперь маскируется штатно, а поле получило кнопку-глаз без рамки для показа и скрытия значения.
- Для `OpenVPN` исправлен контейнерный сценарий: поддержан `auth-user-pass`, корректный разбор `.ovpn`, ожидание готовности туннеля и применение DNS, который пушит VPN-сервер.
- В `Безопасность` сертификаты сохраняются сразу, без отдельной кнопки, а в списках оставлены только `Название` и `Действия`.
- Во вкладке `Личность` и в профильной вкладке `Личность` добавлен режим `Реальный`, сделанный режимом по умолчанию.
- Убран дублирующийся текст-подсказка у режима `Реальный`.
- Для общих `panel/security-frame/tab-pane` добавлен глобальный перенос длинного текста и `pre/code`, чтобы контент не выходил за границы карточек и фреймов.
- Выпадающий список `Режим` в `Личности` сделан компактнее и визуально ближе к селекту `Режим DNS`.

### Core

- Подтверждение условий `Wayfern` переведено в глобальный launcher-scoped режим: после одного принятия они больше не запрашиваются заново для каждого отдельного профиля.
- Приватные корневые сертификаты изолированы только через `Camoufox`; `Wayfern` больше не импортирует их в системное доверие Windows и блокирует запуск с понятным предупреждением.
- При ручной смене движка профиль полностью очищает своё рабочее хранилище, а при удалении профиля launcher дополнительно вычищает связанные локальные данные и привязки сертификатов.
- Fingerprint-модель и валидация профильной модалки расширены режимом `real`, который сохраняет фактические характеристики launcher-окружения как отдельный тип personality-пресета.
- Для `Camoufox` ужесточено подавление cursor/touch overlay: отключены дополнительные Firefox touch/caret prefs и выставлены runtime-флаги `showcursor=false` и `humanize=false`.

### Release

- Весь проект переведён на версию `1.0.11`.
