# Журнал изменений

## 1.0.28

Тестовая версия по проверке работоспособности обновлений

### Обновления

- Усилено сквозное логирование апдейтера: вкладка `Logs` теперь читает persisted runtime log из файла, а `ui/desktop/src-tauri/src/update_commands.rs` и helper-скрипты применения `.msi`/`.zip` пишут туда шаги handoff/apply/relaunch, чтобы финальные события не терялись после закрытия окна апдейтера.
- Переработана надёжность и наблюдаемость локального updater e2e: `scripts/local-updater-e2e.ps1` теперь по умолчанию использует одну попытку (`PublishedRetryCount=0`), выводит явную диагностику timeout/оставшегося бюджета на каждую попытку и предотвращает скрытые длинные хвосты ретраев у глобального дедлайна.
- Усилен fail-fast путь для published updater e2e в `scripts/published-updater-e2e.ps1`: ограничен таймаут MSI helper (`max 120000ms`), добавлена периодическая диагностика `applying` (количество совпадений msiexec + наличие/размер staged MSI лога), и синхронизированы сообщения/бюджет времени при жёсткой остановке для более быстрых и понятных ошибок.
- Расширена телеметрия сборки MSI в `scripts/build-installer.ps1` и телеметрия snapshot-сборки в `scripts/local-updater-e2e.ps1` с таймерами длительности по стадиям, чтобы долгие фазы были наблюдаемыми, а не «тихими».
