# Журнал изменений

## 1.0.24

Тестовая версия по проверке работоспособности обновлений

### Обновления

- Починен MSI-first сценарий обновления: отложенное применение обновления при выходе теперь перед установкой завершает запущенные процессы `Cerbena` из каталога установки, после чего передаёт пакет в `msiexec` и корректно выполняет перезапуск.
- Улучшен published updater e2e: проверочный стенд теперь моделирует установленную MSI-сборку и ждёт фонового этапа `applied_pending_relaunch`, а не обрывает проверку слишком рано.
- Починен сценарий отдельного окна апдейтера `--updater`: после состояния `ready_to_restart` оно теперь автоматически закрывается в auto-режиме, чтобы сработал отложенный handoff в `msiexec`, а локальный release flow больше не зависал на финальном e2e-проходе.

### Релизы

- Публичный GitHub Release переведён на MSI-only контракт: публикуются только `cerbena-browser-<version>.msi`, `checksums.txt`, `checksums.sig` и `release-manifest.json`.
- В `release.ps1` добавена очистка лишних GitHub assets при публикации, чтобы старые `zip`, `setup.exe` и отдельный `updater.exe` больше не оставались на странице релиза.

### Безопасность

- Убран из отслеживания репозитория `ui/desktop/src-tauri/Cargo.lock`, из-за которого Dependabot поднимал нерелевантные для Windows-релизов advisory по GTK/GLib-стеку Tauri.
- Локальные vulnerability gates больше не аудируют вложенный desktop lockfile и не требуют его присутствия в version-sync контракте.
