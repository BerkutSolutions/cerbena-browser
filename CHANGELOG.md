# Журнал изменений

## 1.0.16

### Стабильность

- Улучшена диагностика launcher-апдейтера: при ошибках запроса к `GitHub Releases API` Cerbena теперь пишет в runtime-логи URL, `User-Agent`, HTTP-статус, `x-github-request-id`, `content-type` и сокращённое тело ответа, чтобы отличать реальные `403` GitHub от локальных проблем конкретной машины без ручной отладки в коде.
- В боковое меню добавлена вкладка `Логи` с live-потоком runtime-сообщений лаунчера: в ней теперь видны предупреждения route/runtime, сообщения `KeePassXC`, диагностика апдейтера и необработанные frontend-ошибки/`unhandledrejection`, которые сразу попадают в общий launcher log.
