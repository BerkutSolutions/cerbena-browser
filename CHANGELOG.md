# Журнал изменений

## 1.0.3 — KeePassXC, launcher и документация

### Повышение безопасности

- Интеграция `KeePassXC` для профилей `Wayfern` переведена на launcher-side native messaging bridge: manifest и registry-ветки подготавливаются автоматически, origin расширения определяется по фактическому runtime-id профиля, а соединение больше не должно блокироваться ошибкой `Access to the specified native messaging host is forbidden`.

### Стабильность

- При завершении launcher теперь дополнительно выполняется shutdown всех профилей с дочерними процессами не только по активной карте сессий, но и по всем profile-scoped runtime-каталогам, чтобы браузерные процессы не оставались висеть после закрытия оболочки.
- Контрактные версии в тестах и runtime-значениях выровнены под релизную линию `1.0.3`.

### Документация

- В wiki добавлена отдельная страница по интеграции `KeePassXC` с описанием native messaging bridge, модели разрешений, runtime-origin и причин, по которым интеграция не нарушает profile isolation.
