# Журнал изменений

## 1.0.18

### UI

- Во вкладке `Настройки -> Основные` добавлен новый фрейм `Состояние средств` с индикаторами доступности установленных компонентов.
- Во вкладке `Настройки -> Ссылки` наверх перенесены `Поисковик по умолчанию` и `Стартовая страница`.
- Блок `Обновления лаунчера` приведён к тем же размерам и стилю рамки, что и `Состояние устройства`.

### Утилиты

- Для управляемых runtime-компонентов добавена проверка статуса, отображение установленной версии и кнопка `Скачать`, если компонент отсутствует.
- Для `Docker Desktop` сохранён ручной сценарий установки через переход на внешний сайт, как и в существующем модальном потоке.
- В статусах учтены как локально установленные средства, так и зависимости, поднимаемые через `Docker`.

### Обновления

- Исправлена гиперссылка `Открыть релиз`: теперь она открывает точную страницу GitHub release для версии запущенного приложения.
- Улучшена подготовка UI к диагностике обновлений за счёт явного разделения статуса инструментария и статуса обновления лаунчера.

### Безопасность

- Из репозитория убран встроенный приватный release-ключ: проверочный публичный материал теперь коммитится отдельно, а приватные signing-секреты подаются только оператором во время выпуска.
- Для Windows release flow добавлены обязательные проверки `Authenticode` и отдельной подписи `checksums`, чтобы публикация падала при отсутствующей или невалидной подписи.

### Установщик и релизы

- Добавлен полноценный `MSI`-поток сборки для Windows с `UpgradeCode`, локальной установкой в профиль пользователя, ярлыками, browser registration и launcher-owned cleanup при удалении.
- Windows release layout стал `MSI-first`: `release-manifest.json` и `checksums.txt` теперь публикуют `MSI` как основной установочный артефакт, а `.exe` остаётся ручным fallback-сценарием.
- Updater научился различать `portable zip`, `direct MSI` и `manual installer` path, сохраняет причину выбора артефакта и диагностирует handoff через `msiexec` без непрозрачного извлечения внутреннего payload.

### Документация и тесты

- Добавлены регрессионные проверки для MSI-first release contract, выбора updater-артефакта и stale handoff/relaunch state.
- Обновлены операторские инструкции по выпуску, локальной проверке подписи `checksums`, верификации `Authenticode` и восстановлению после неуспешного MSI-обновления.
