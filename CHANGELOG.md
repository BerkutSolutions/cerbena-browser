# Журнал изменений

## 1.0.10 — маршрутизация ссылок, single-instance launcher и строгие профили

### UI

- Для профилей с импортированными приватными корневыми сертификатами запуск `Wayfern` теперь жёстко блокируется понятным предупреждением вместо молчаливого автопереключения движка.
- Подтверждение условий `Wayfern` переведено в глобальный launcher-scoped режим: после одного принятия правила больше не запрашиваются заново для каждого отдельного Chromium/Wayfern-профиля.
- Во вкладке `Безопасность` сертификаты сохраняются сразу, без отдельной кнопки `Сохранить`, а в списках оставлены только `Название` и `Действия`.
- В `Camoufox` добавлены дополнительные desktop-safe prefs против touch/caret overlay и курсорных колец: отключены accessible caret, pointer-message dispatch, direct manipulation, экранная клавиатура, а также явные runtime-флаги `showcursor=false` и `humanize=false`.
- Добавлены стартовые модальные окна для сценария браузера по умолчанию: пользователь может включить проверку на старте, а если `Cerbena` уже является браузером по умолчанию для `HTTP/HTTPS`, сразу выбрать профиль по умолчанию для раздела «Ссылки».
- В `Настройки -> Ссылки` добавлен чекбокс проверки браузера по умолчанию с сохранением отказа между перезапусками и после обновлений.
- В `Настройки -> Общие` рядом с блоком обновлений добавлен чекбокс `Сворачивать в трей вместо закрытия`.
- Модальное окно выбора профиля для открытия ссылки выровнено: кнопки теперь тянутся на всю ширину контейнера и центрируются без «лесенки».
- В настройки и внутренний роутинг ссылок добавлены дополнительные системные типы и ассоциации: `IRC`, `MAILTO`, `MMS`, `NEWS`, `NNTP`, `SMS`, `SMSTO`, `SNEWS`, `TEL`, `URN`, `WEBCAL`, а также файловые типы `.mht`, `.mhtml`, `.pdf`, `.shtml`, `.svg`, `.xhtml`, `.xhy`, `.xht`.
- Встроены строгие профили `Discord` и `Telegram`: они запускаются как app-window без свободной адресной строки и ограничены только своими разрешёнными доменами.
- В модалку профиля на вкладку `Основная` добавлен чекбокс `Одностраничный`: для `Wayfern` он превращает профиль в строгое app-окно по домену стартовой страницы и скрывает выбор поисковика по умолчанию.

### Core

- Профильные приватные `CA` теперь реально изолированы только через `Camoufox`: назначенные сертификаты хранятся в managed-хранилище launcher'а и применяются внутри профильного Firefox/NSS-контура без импорта в системное доверие Windows.
- `Wayfern` больше не импортирует сертификаты в Windows trust store и не стартует с назначенными приватными корневыми сертификатами: backend останавливает запуск с ошибкой `profile.security.wayfern_certificates_not_supported`.
- При ручной смене движка профиль полностью очищает своё рабочее хранилище (`engine-profile`, runtime/policy-артефакты и связанные временные данные), чтобы данные старого браузера не смешивались с новым движком.
- При удалении профиля launcher теперь дополнительно вычищает связанные profile-scoped записи из identity/network/sync/link-routing/network-sandbox/extension-assignment/security-store и удаляет привязки сертификатов вместе с локальными данными профиля.
- Исправлена контейнерная интеграция `OpenVPN`: launcher теперь создаёт и монтирует `auth-user-pass` credentials-файл для контейнерного рантайма, а helper-контейнер не поднимает локальный `SOCKS`-вход до реального `Initialization Sequence Completed`, что убирает сбой `openvpn profile requests auth-user-pass; set username/password fields` и сокращает риск преждевременного прямого трафика до поднятия туннеля.

- Добавлен persist-store shell-настроек (`shell_preference_store.json`) для хранения флагов проверки браузера по умолчанию, режима трея и пользовательских отказов.
- В Windows-регистрацию установки добавлены browser capabilities и `ProgId` для `http/https`, чтобы `Cerbena` корректно определялся в проверке браузера по умолчанию и оставался доступным в системных настройках ассоциаций.
- Реализован системный трей с действиями восстановления окна и полного выхода из приложения.
- Добавлен межпроцессный handoff внешних ссылок: повторный запуск `Cerbena` с URL теперь передаёт ссылку уже открытому экземпляру лаунчера вместо запуска второго окна приложения.
- Для уже запущенного профиля внешний URL теперь отправляется в существующую сессию браузера без повторного полного launch-cycle профиля.
- Обработка handoff внешних ссылок перенесена на самый ранний этап старта приложения, до поднятия Tauri shell, чтобы исключить визуальное появление дублирующегося лаунчера при открытии `http/https` из других приложений.
- Лаунчер переведён в жёсткий single-instance режим: одновременно активен только один экземпляр, а повторный запуск передаёт активацию или ссылку уже открытому окну.

### Lifecycle

- Закрытие главного окна теперь учитывает настройку трея: при включённой опции окно скрывается в трей, при выключенной и отсутствии сохранённого отказа приложение предлагает включить этот режим вместо завершения.
- Сохранённые отказы по стартовой проверке браузера и предложению про трей переживают обновление приложения и повторно не навязываются.
- Проверка и подтверждение условий `Wayfern` перенесены на ранний стартовый preflight для существующих Chromium/Wayfern-профилей, чтобы первое открытие внешней ссылки не блокировалось ошибкой `wayfern_terms_not_acknowledged`.
- Глобальный профиль по умолчанию для ссылок по умолчанию применяется только к `HTTP` и `HTTPS`; остальные поддерживаемые типы остаются отключёнными, пока пользователь не привяжет их явно.

### Release

- Весь проект переведён на версию `1.0.10`, включая desktop shell, workspace crates, smoke/UI fallbacks и release-sensitive тестовые ожидания.
- Fallback Windows installer перед обновлением теперь принудительно завершает уже запущенные процессы установленной `Cerbena`, чтобы новая версия действительно запускалась после установки и не оставляла пользователя в старом процессе из трея.
- Документация обновлена под `1.0.10`: описаны новые встроенные профили, одностраничный режим профиля и расширенный список поддерживаемых ссылок и файловых ассоциаций.
