Безопасность Cerbena Browser строится вокруг zero-trust, изоляции профилей и модели сети с запретом по умолчанию.

## Базовые гарантии

- backend верифицирует каждое критичное действие;
- межпрофильный доступ запрещен по умолчанию;
- route policy и DNS policy применяются в контексте профиля;
- журнал аудита фиксирует чувствительные операции;
- `auto-update` выключен по умолчанию.

## Ключевые механизмы

- парольная блокировка профиля;
- `ephemeral`-режим;
- шифрование данных профиля на диске;
- зашифрованное хранение чувствительных данных оболочки вроде `Sync`, маршрутизации ссылок и реестра сессий запуска;
- защищенные снимки Sync, которые теперь содержат реальный профильный материал для восстановления вместо демонстрационной нагрузки, с шифрованием резервного контейнера и проверкой целостности перед восстановлением;
- зашифрованное хранение глобального security-состояния оболочки, включая стартовую страницу, управляемые сертификаты и реестр DNS-блоклистов, вместо открытого файла `_global-security.json`;
- защита запуска чувствительных профилей: перед стартом требуется разблокировка, опасные сочетания `доступа расширений к системе` и `KeePassXC` блокируются, для политики `Maximum` запрещен запуск с включенными расширениями, а прямое копирование cookie в защищенные профили не допускается;
- контур состояния устройства, который перед запуском защищенного профиля проверяет сигналы хоста и может предупредить, потребовать подтверждение или отказать в запуске в зависимости от политики профиля и тяжести риска;
- реестр сессий запуска с контрольным файлом рабочей области, который не позволяет доверять любому процессу только по совпадению каталога профиля;
- kill-switch;
- сервисная и доменная фильтрация;
- аварийная и выборочная очистка;
- аварийная очистка с политикой защищенных сайтов, точным сохранением по доменам для хранилищ куки и истории у движков семейств Хромиум и Файрфокс, а также Windows-оверлеем вокруг изолированного окна браузера;
- проверка подписей release-артефактов.

## Сетевые риски, которые закрывает продукт

- обход обязательного VPN;
- утечки DNS;
- смешение политик разных профилей;
- использование глобальных блокировок без явной видимости причины.
