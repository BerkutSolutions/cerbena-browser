Модуль `Sync` в Cerbena Browser отвечает за самостоятельно размещаемую синхронизацию, снимки и восстановление состояния профиля.

## Базовые элементы

- элементы управления sync на уровне профиля;
- проверка доступности endpoint;
- список конфликтов;
- снимки и поток восстановления;
- модель `E2E encryption`.

## Где это находится в текущем UI

- профиль хранит собственные параметры sync;
- централизованное управление находится в `Settings > Sync`;
- там же доступны статус, доступность endpoint, снимки и восстановление для выбранного профиля.

## Что есть в desktop-контуре

- сохранение и чтение параметров sync профиля;
- создание снимка из desktop UI;
- восстановление с проверкой целостности;
- backend-обновление статуса здоровья соединения;
- E2E-защита полезной нагрузки через случайный ключ данных и ключ, обернутый через `AEAD`;
- резервное дешифрование старого конверта, чтобы ранее созданные sync-данные оставались читаемыми после обновления усиления защиты;
- транспортная защита с проверкой TLS как структуры протокола и окном недавних одноразовых значений `nonce` против сценариев повторного воспроизведения и понижения версии.

## Рекомендации

- включайте `sync` только там, где он действительно нужен;
- храните URL сервера и ключи отдельно от профилей, если того требует политика;
- проверяйте восстановление на тестовом профиле до применения в рабочем контуре.
