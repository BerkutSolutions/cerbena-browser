Модуль `Identity` управляет тем, как профиль выглядит для сайтов и remote services.

## Режимы

- `Автоматически`: генерация правдоподобного профиля под выбранную платформу без ручного редактирования каждого поля.
- `Ручной`: явное редактирование полей с поддержкой генерации и шаблонов.

## Что изменилось в текущем UI

- В режиме `Автоматически` кнопка ручной генерации и шаблоны не показываются.
- В режиме `Автоматически` личность генерируется заново на каждую сессию профиля.
- В режиме `Ручной` появляется кнопка `Сгенерировать` и список шаблонов, который реально подставляет значения в форму.
- Поддерживаются дополнительные платформенные шаблоны `Debian`, `Ubuntu` и `Windows 8`.

## Группы параметров

- `UA`, платформа, версия, brand и vendor;
- hardware: CPU threads, touch points, device memory;
- screen/window: width, height, DPR, inner/outer sizes, coordinates;
- locale/timezone/geo;
- `WebGL`, `Canvas`, fonts;
- audio и battery.

## Validator согласованности

Перед сохранением и запуском backend проверяет:

- соответствие platform и `UA`;
- реалистичность screen/window-комбинаций;
- связность locale/timezone/geo;
- допустимость JSON-полей и числовых значений;
- отсутствие очевидных конфликтов с network policy.

## Когда использовать `Автоматически`

- для типовых рабочих профилей;
- когда важнее быстро получить реалистичный baseline;
- когда не нужен ручной контроль каждого поля.

## Когда использовать `Ручной`

- для исследовательских и совместимых тестовых сценариев;
- при миграции существующего профиля;
- когда нужна точная имитация платформы или браузерной среды.
