## Текущие ограничения

- часть runtime-потока ориентирована прежде всего на Windows-host.
- `desktop UI` и `docs-site` собираются отдельными инструментальными контурами.
- route runtime зависит от managed binaries и их локальной доступности.
- некоторые `VPN`/`proxy`-цепочки ограничены поддерживаемыми backend-преобразованиями.

## Осознанные решения

- `auto-update` выключен по умолчанию;
- UI не пытается быть security boundary;
- release-gates смещены в сторону проверок и воспроизводимости, а не максимальной "магии" на клиенте.
