## Текущие ограничения

- часть runtime-потока ориентирована прежде всего на Windows-хост.
- `desktop UI` и сайт документации собираются отдельными инструментальными контурами.
- маршрутный runtime зависит от управляемых бинарных файлов и их локальной доступности.
- некоторые `VPN`/`proxy`-цепочки ограничены поддерживаемыми backend-преобразованиями.

## Осознанные решения

- `auto-update` выключен по умолчанию;
- UI не пытается быть границей безопасности;
- релизные проверки смещены в сторону воспроизводимости и верификации, а не максимальной "магии" на клиенте.
