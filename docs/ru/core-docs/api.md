Cerbena Browser предоставляет локальные поверхности автоматизации, но все они подчиняются zero-trust-валидации.

## Локальный API

Назначение:

- открытие ссылок через профиль по умолчанию;
- запуск рабочих сценариев автоматизации;
- интеграция с внешними desktop-инструментами и локальными агентами.

## MCP

`MCP`-слой предоставляет инструментальную поверхность для работы с профилями и политиками, но:

- область действия ограничивается backend-контрактом;
- каждая чувствительная операция должна быть авторизована;
- критичные действия журналируются.

## Guardrails

- нельзя доверять только UI-проверкам;
- нельзя выполнять операции профиля без явного контекста профиля;
- нельзя расширять область действия в обход backend-авторизации.

Для подробностей по интеграции используйте раздел [Интеграции локального API и MCP](../integration-docs/local-api-and-mcp.md).
