Cerbena Browser предоставляет локальные automation surfaces, но все они подчиняются zero-trust-валидации.

## Локальный API

Назначение:

- открытие ссылок через default profile;
- запуск рабочих сценариев автоматизации;
- интеграция с внешними desktop-инструментами и локальными агентами.

## MCP

`MCP`-слой предоставляет инструментальную поверхность для работы с профилями и политиками, но:

- scope ограничивается backend-контрактом;
- каждая чувствительная операция должна быть авторизована;
- критичные действия журналируются.

## Guardrails

- нельзя доверять только UI-проверкам;
- нельзя выполнять profile-операции без явного profile context;
- нельзя расширять scope в обход backend authorization.

Для подробностей по интеграции используйте раздел [Интеграции local API и MCP](../integration-docs/local-api-and-mcp.md).
