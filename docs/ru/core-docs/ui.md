Интерфейс Cerbena Browser построен как локальная оболочка над backend-командами `Tauri`. Он не считается доверенной границей, но отражает фактическое состояние профилей, сетевых политик, runtime-процессов и защитных механизмов.

## Реальные разделы оболочки

- `Home`: главная рабочая поверхность для создания профилей, сводных метрик, массовых действий, запуска, остановки, импорта, экспорта и статусов.
- `Extensions`: библиотека расширений, импорт из ссылок и архивов, назначение профилям, совместимость по движку.
- `Security`: пользовательские сертификаты и глобальные защитные параметры оболочки.
- `Identity`: режимы `Автоматически` и `Ручной`, генерация личности, шаблоны и проверка согласованности.
- `DNS`: режимы DNS, таблица уровней политики, blocklists, список запрещенных суффиксов, каталог сервисов и доменные ограничения.
- `Network`: шаблоны соединений, режим маршрутизации, глобальная VPN-политика, глобальная политика сетевой изоляции и проверки доступности узлов.
- `Traffic`: журнал решений шлюза трафика, ручные блокировки доменов и диагностика маршрутов.
- `Settings`: глобальные настройки оболочки с вкладками `General`, `Links`, `Sync` и управлением обновлениями.
- `Updater`: отдельное окно `cerbena-updater.exe` для доверенного потока обновления, предпросмотра и безопасной передачи на установку.

## Встроенные профили и ссылки

- `Home` сразу содержит шесть встроенных профилей: `Chromium Default`, `Firefox Default`, `Chromium Private Memory`, `Firefox Private Memory`, `Discord` и `Telegram`.
- `Discord` и `Telegram` — это строгие `Wayfern`-профили в формате окна приложения с фиксированными allowlist-правилами и без свободной адресной строки.
- Во вкладке `General` модалки профиля есть чекбокс `Одностраничный` для пользовательских строгих `Wayfern`-профилей. В этом режиме стартовая страница остаётся обязательной, а поле поисковика по умолчанию скрывается.
- `Settings > Links` поддерживает маршрутизацию для `HTTP`, `HTTPS`, `FTP`, `MAILTO`, `IRC`, `MMS`, `NEWS`, `NNTP`, `SMS`, `SMSTO`, `SNEWS`, `TEL`, `URN`, `WEBCAL`, `TG`, `Discord`, `Slack`, `Zoom`, а также файловые ассоциации для `.htm`, `.html`, `.shtml`, `.mht`, `.mhtml`, `.pdf`, `.svg`, `.xht`, `.xhtml` и `.xhy`.
- Только `HTTP` и `HTTPS` автоматически наследуют глобальный профиль по умолчанию. Для остальных поддерживаемых типов нужна явная привязка.

## Ключевые принципы интерфейса

- Любое чувствительное действие подтверждается backend-валидацией.
- Профили редактируются через модальное окно с левой вертикальной навигацией по секциям `General`, `Identity`, `VPN`, `DNS`, `Extensions`, `Security`, `Sync`, `Advanced`.
- Главный экран `Network` управляет глобальными шаблонами маршрутов и фреймом `Сетевая изоляция профилей`. Этот фрейм показывается только при включенном глобальном VPN, потому что только тогда у launcher есть активный глобальный маршрут для оценки.
- Вкладка `VPN` в модалке профиля управляет шаблоном конкретного профиля и фреймом `Сетевая изоляция профиля`. Этот фрейм учитывает выбранный маршрут и предлагает только реально совместимые режимы изоляции.
- Во вкладке `Трафик` таблица стала плотнее и понятнее: колонка запроса специально сделана шире, колонка ответа компактнее, а меню действий и всплывающие слои рисуются поверх интерфейса и больше не обрезаются строками таблицы.
- Для долгого запуска профиля показывается временная модалка хода запуска, чтобы пользователь видел этапы подготовки профиля, маршрутного runtime, контейнерной среды, вспомогательного образа и браузерного движка.
- Сетевые и runtime-статусы не считаются достоверными без backend-проверки.

## Базовый рабочий сценарий

1. Создать профиль на `Home`.
2. Открыть модалку профиля и настроить `Identity`, `VPN`, `DNS`, `Extensions` и `Security`.
3. При необходимости настроить глобальный маршрут по умолчанию и глобальную изоляцию в `Network`.
4. Проверить `Settings` для стартовой страницы, поисковика по умолчанию, ссылок, sync и политики обновлений.
5. Запустить профиль и наблюдать `Traffic`, если маршрут заблокирован или работает нестабильно.

## Что смотреть при неполадках

- `Traffic`, если нужно понять причину блокировки, задержку маршрута или решение kill-switch.
- `Network`, если проблема связана с шаблонами, стратегией изоляции, емкостью контейнерной среды или доступностью вспомогательного runtime.
- `DNS`, если проблема связана с уровнем политики, blocklists, запрещенными суффиксами или сервисными ограничениями.
- `Settings > Sync`, если проблема касается синхронизации, доступности endpoint или snapshot-сценариев.
- `Updater`, если проблема касается проверки релиза, доверенного скачивания, сверки сумм или состояния `версия уже актуальна`.
- operator/runtime-диагностику, если проблема связана с подготовкой бинарных файлов, установкой движка, потоком установщика или маршрутным runtime.
