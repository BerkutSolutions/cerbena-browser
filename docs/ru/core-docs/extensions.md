Cerbena Browser хранит расширения в общей библиотеке, но применяет их к профилям изолированно.

## Что поддерживается

- импорт из локального архива через системный file picker;
- импорт по поддерживаемой store-ссылке;
- извлечение реального display name, version, engine scope и icon из пакета;
- назначение на выбранные профили;
- enable/disable и удаление.

## Изоляция

- installation state и local data являются per-profile;
- совместимость проверяется по engine (`Wayfern` / `Camoufox`);
- extension policy не должна обходить route policy или DNS-ограничения профиля.

## Поток работы

1. Добавить расширение в библиотеку через ссылку или архив.
2. Проверить metadata, icon и engine compatibility.
3. Назначить его на профиль.
4. Запустить профиль и дождаться first-launch install flow.
5. При необходимости отключить, переустановить или удалить расширение.

## Что важно помнить

- Firefox-ориентированные расширения не должны назначаться Chromium-профилю и наоборот;
- UI должен показывать имя расширения как основной идентификатор, а не raw id;
- policy enforcement должен оставаться backend-driven.
