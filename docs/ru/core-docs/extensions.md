title: Расширения

Cerbena Browser хранит расширения в общей библиотеке, но применяет их к профилям изолированно.

## Что поддерживается

- импорт из локального архива через системный выбор файла;
- импорт по поддерживаемой ссылке магазина;
- экспорт библиотеки в выбранную папку либо как файл со ссылками, либо как папку с архивами пакетов;
- импорт ранее экспортированного файла со ссылками или архивной папки из выбранного каталога;
- извлечение реального имени, версии, области движка и значка из пакета;
- ручные теги для расширений и фильтрация библиотеки по тегам без чувствительности к регистру;
- назначение на выбранные профили;
- включение, выключение и удаление.

## Изоляция

- состояние установки и локальные данные являются профильно-специфичными;
- совместимость проверяется по engine (`Wayfern` / `Camoufox`);
- политика расширений не должна обходить политику маршрутизации или DNS-ограничения профиля.

## Поток работы

1. Добавить расширение в библиотеку через ссылку или архив.
2. Проверить метаданные, значок и совместимость по движку.
3. При необходимости добавить теги и отфильтровать библиотеку по ним.
4. Назначить его на профиль.
5. Запустить профиль и дождаться первичной установки при первом запуске.
6. При необходимости экспортировать или повторно импортировать библиотеку через действия `Файл` / `Архив`.

## Что важно помнить

- Firefox-ориентированные расширения не должны назначаться Chromium-профилю и наоборот;
- UI должен показывать имя расширения как основной идентификатор, а не сырой идентификатор;
- принуждение политик должно оставаться на стороне backend.
