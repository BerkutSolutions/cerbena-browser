title: Маршрутизация и маршрутный runtime

Сетевой контур Cerbena Browser строится вокруг профильно-специфичной политики маршрутизации, traffic gateway и управляемого маршрутного runtime.

## Поддерживаемые режимы

- `direct`
- `proxy`
- `vpn`
- `tor`
- `hybrid`

Дополнительно доступны глобальные настройки `global VPN` и `block without VPN`.

## Шаблоны соединений

Шаблоны соединений описывают один или несколько узлов:

- `proxy` (`http`, `socks4`, `socks5`, `shadowsocks`, `vmess`, `vless`, `trojan`);
- `vpn` (`wireguard`, `openvpn`, `amnezia`);
- `tor` (`none`, `obfs4`, `snowflake`, `meek`).

Шаблон может быть привязан к профилю или назначен как глобальный маршрут по умолчанию.

## Маршрутный runtime

Если маршруту недостаточно простого прямого подключения, Cerbena Browser поднимает локальный runtime:

- `sing-box` для цепочек и `v2ray`-совместимых протоколов;
- `openvpn` для `openvpn`;
- `amneziawg` для нативного `AmneziaWG`;
- `tor` и подключаемых транспортных модулей для `TOR`.

Runtime управляется автоматически при запуске и остановке профиля.

## Traffic gateway

Локальный шлюз:

- принимает профильный трафик;
- применяет kill-switch, DNS-правила, сервисные и доменные ограничения, а также ручные блокировки пользователя;
- пишет журнал решений в `Traffic`;
- умеет работать через runtime `SOCKS5`-узел или напрямую через прокси.

## Kill-switch

Kill-switch блокирует трафик, если:

- обязательный VPN/runtime-маршрут недоступен;
- `TOR`-шаблон невалиден или мост не отвечает;
- глобальная политика требует VPN, но активной runtime-сессии нет.

Никакая поздняя логика политик не должна ослаблять это правило.

## Что проверять при проблемах

1. Выбран ли корректный шаблон.
2. Поднят ли маршрутный runtime.
3. Не включен ли глобальный `block without VPN`.
4. Есть ли сообщение в `Traffic` или operator/runtime diagnostics о причине блокировки.
