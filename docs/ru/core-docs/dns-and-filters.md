Cerbena Browser комбинирует DNS, привязанный к профилю, blocklists, каталог сервисов и доменные правила.

## DNS-режимы

- `Системный`
- `Ручной`

Для `Ручной` доступны обычные адреса резолверов, а также ограничения `DoH` и `DoT`, если они заданы профилем.

## Что находится на DNS-странице

- глобальные DNS blocklists;
- каталог сервисов в виде аккордеона;
- blacklist доменных суффиксов;
- привязанные к профилю правила разрешения и запрета доменов, а также выбранные сервисы.

`DNS blocklists` и `Каталог сервисов` в текущем UI отображаются как аккордеоны, чтобы страница оставалась компактной.

## Источники блокировок

- profile `domain_denylist`;
- profile `selected_blocklists`;
- profile `selected_services`;
- глобальные DNS blocklists;
- глобальный список запретов по доменным суффиксам;
- ручные блокировки пользователя из `Traffic`.

## Правила разрешения конфликтов

1. Жесткие ограничения важнее всего.
2. Явный запрет домена сильнее разрешения сервиса.
3. Явное разрешение домена сильнее блокировки категории, если нет жесткого ограничения.
4. Исключение переопределяет базовое правило своего слоя.
5. Если конфликт не разрешен, действует `deny-by-default`.

## Практика эксплуатации

- сначала определите политику маршрутизации;
- затем подключите DNS-серверы и blocklists;
- после этого включайте сервисные категории и точечные исключения;
- при спорном результате смотрите `Traffic` и код причины backend-решения.
