title: Zero-trust и backend-принуждение
sidebar_position: 6

Cerbena Browser исходит из того, что `UI` может ошибаться, отставать по состоянию или быть обойден.

## Следствия

- backend повторно валидирует каждое чувствительное действие;
- проверки авторизации и области действия обязательны для local API и `MCP`;
- запуск, очистка, синхронизация, сетевая политика и операции с расширениями не должны доверять только клиентской логике;
- журнал аудита обязателен для критичных операций.

## Что это дает

- защиту от модели "безопасность только в UI";
- воспроизводимое поведение при автоматизации;
- предсказуемые релизные проверки и трассируемость.
