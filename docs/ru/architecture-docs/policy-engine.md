Policy engine Cerbena Browser должен принимать детерминированные и объяснимые решения.

## Порядок вычисления

1. жесткие ограничения
2. политика маршрутизации
3. DNS-политика
4. доменные и сервисные политики
5. исключения

## Жесткие ограничения

- deny при отсутствии контекста профиля;
- deny при недоступном обязательном `VPN`;
- deny при нарушении `TOR-only`;
- deny при невалидном или незагруженном пакете политик.

## Объяснимость

Каждое решение должно иметь:

- итоговое действие;
- выбранный маршрут;
- сработавшие правила;
- код причины;
- итоговый DNS;
- след конфликтных переопределений.
